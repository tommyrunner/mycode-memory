# MyCode Memory

这个 skill 维护项目内记忆文件：`.mycode-memory/memory.md`，核心用途是记住用户在具体项目中的个人编码习惯。

它不是常规编码规则或通用最佳实践的生成器，只记录用户明确表达过的习惯、Codex 在对话中反馈过并形成结论的项目事实，以及当前项目中确实能验证的稳定信息。

## 触发条件

本 skill 只有两个职责：读取记忆、更新记忆。

### 读取记忆

当用户提出和项目代码相关的工作时，使用本 skill 的读取记忆能力，例如：

- 写代码
- 改代码
- 修 bug
- 重构
- 阅读项目
- 添加功能
- 优化组件
- 调整文件结构
- 排查项目问题

读取记忆时只允许检查、创建和读取 `.mycode-memory/memory.md`，并在后续编码中遵循其中内容。普通代码任务结束后不要更新记忆文件。

### 更新记忆

只有用户明确调用 `mycode-memory`，并且表达更新、刷新、整理、细化记忆时，才使用更新记忆能力，例如：

- `$mycode-memory 更新记忆`
- `$mycode-memory 刷新记忆`
- `$mycode-memory 根据当前对话更新 memory.md`
- `使用 mycode-memory 更新项目记忆`

没有明确调用 `mycode-memory` 时，即使用户说到编码习惯，也不要修改 `.mycode-memory/memory.md`。

## 记忆位置

```text
.mycode-memory/
  memory.md
```

## 普通编码模式

当用户提出项目代码相关工作时：

1. 检查项目根目录是否存在 `.mycode-memory/memory.md`。
2. 如果不存在，从本 skill 目录运行 `node scripts/init-memory.js` 创建。
3. 读取 `.mycode-memory/memory.md`。
4. 工作时遵循其中记录的用户习惯和项目记忆。
5. 普通编码任务结束后不要自动更新 `memory.md`。

优先级：

1. 用户当前明确要求。
2. 项目现有行为和局部代码风格。
3. `.mycode-memory/memory.md`。
4. 通用工程判断。

## 记忆刷新模式

只有用户明确调用 `mycode-memory`，并要求刷新、总结或更新记忆时使用。

一旦进入本模式，“更新记忆”“刷新记忆”“总结记忆”等表达都默认表示：主动回顾当前完整对话上下文，提取用户习惯、Codex 反馈结论和项目事实，再与已有 `memory.md` 融合更新。用户不需要额外说明“请把当前回话全过一遍”。

示例：

- `$mycode-memory 刷新记忆`
- `$mycode-memory 根据上面的对话更新 memory.md`
- `使用 mycode-memory 总结上述对话并更新项目记忆`

刷新流程：

1. 确保 `.mycode-memory/memory.md` 存在；不存在时用 `scripts/init-memory.js` 创建。
2. 编辑前先完整读取已有 `memory.md`。
3. 主动回顾当前完整对话上下文，而不只看最后一条用户消息。
4. 只从以下来源提取记忆：
   - 用户在对话中明确说过的偏好、要求、习惯。
   - Codex 在对话中明确反馈过的项目事实或执行结论。
   - 当前项目中能直接验证的稳定事实，例如公共文件、目录结构、脚本命令。
5. 如果某个章节没有对应依据，保持“暂无记录”，不要补通用规范。
6. 更新时必须融合已有记忆和新信息，进行改写、合并、修正或替换，不要一直追加。
7. 从局部模块得到的信息，要抽象成项目级记忆，不要写“当前模块/当前页面/本次观察”。
8. 组件化记忆重点记录用户在界面绘制时希望如何拆组件、如何分层、哪些内容独立成组件，以及对应实例。
9. 文件结构记忆分两类：
   - `项目基本信息 / 文件结构`：用序号写项目整体结构；公共重要文件可写具体路径，业务模块只写最上层目录或主要入口。
   - `文件结构习惯`：写以后新增或修改模块时应该采用的结构习惯。
10. 测试与验证只记录用户明确要求或对话中已执行/反馈的验证习惯。例如用户说“不需要 build 打包”，就记录为习惯；如果没说，就写“暂无记录”。

## 记忆编辑规则

- `memory.md` 全程使用中文记录。
- 不记录密钥、token、密码、私钥、用户隐私或敏感业务数据。
- 不把通用最佳实践写进记忆，除非用户明确要求或项目证据支持。
- 不把推测写成记忆。
- 不复制临时聊天原文，要沉淀为简短、明确、可执行的记忆。
- 不使用“当前模块”“当前页面”“当前这次观察”等临时上下文表述。
- 更新已有内容时要去重、合并同类项、修正过时内容；记忆文件应该越来越准，而不是越来越长。
- 执行“更新记忆”时必须主动检索当前对话中所有相关上下文，不要只总结最后一条消息。

## 刷新时检查章节

- 项目基本信息
- 注释习惯
- 打印与调试习惯
- 界面组件化习惯
- 文件结构习惯
- 测试与验证习惯
- 记忆刷新规则
